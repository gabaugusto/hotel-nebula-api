# Guia de Contribuição

Obrigado por querer contribuir com o **Hotel Nebula API**.

Este projeto é educacional e voltado para iniciantes, então priorizamos contribuições simples, claras e didáticas.

## Como contribuir

1. Faça um fork do repositório.
2. Crie uma branch descritiva:
   - `feat/nome-da-melhoria`
   - `fix/nome-do-ajuste`
3. Faça alterações pequenas e focadas.
4. Abra um Pull Request explicando:
   - objetivo da mudança
   - arquivos alterados
   - como testar

## Sugestões de contribuição

- Melhorias de documentação (`README`, exemplos, tutoriais)
- Novos endpoints simples e bem documentados
- Correções de bugs com impacto claro
- Exemplos de requests para Postman/Insomnia
- Pequenos refactors que aumentem legibilidade

## Boas práticas

- Mantenha consistência de nomenclatura entre Java e Kotlin.
- Evite misturar muitas mudanças no mesmo PR.
- Não inclua segredos, tokens ou senhas reais.
- Prefira mensagens de commit objetivas.

## Checklist antes do PR

- [ ] Código compila no módulo alterado
- [ ] README/guia atualizado (se necessário)
- [ ] Coleção de testes atualizada (se endpoint mudou)
- [ ] Sem arquivos temporários ou dados sensíveis

## Observação

Este projeto **não é destinado a produção** e pode não conter todos os requisitos de segurança e robustez exigidos em ambientes profissionais.
